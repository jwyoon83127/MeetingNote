# 📋 사내 회의록 시스템 작업 계획서 (MeetingHub)

> 작성일: 2026년 3월 12일  
> 프로젝트명: 사내 회의록 관리 시스템 (MeetingHub)  
> 목표: 주간 회의를 체계적으로 관리하고, 안건·검토의견·결과를 통합 관리하는 사내 시스템 구축

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적
- 주간 회의의 안건·검토의견·결과를 일원화된 시스템에서 관리
- 회의록 작성·검색·공유를 효율화하여 업무 연속성 확보
- 결정 사항 및 액션 아이템의 추적 관리로 실행력 향상

### 1.2 핵심 요구사항
| 구분 | 내용 |
|------|------|
| 주기 | 주간 단위 회의 진행 |
| 안건 관리 | 회의 전 안건 등록 및 사전 공유 |
| 검토의견 | 각 안건별 참석자 의견 기록 |
| 결과 기록 | 안건별 결정 사항 및 후속 조치 |
| 회의록 정리 | 자동 포맷팅 및 배포 기능 |

---

## 2. 시스템 구성 (화면 설계)

### 2.1 전체 화면 구조

```
┌─────────────────────────────────────────────────────┐
│  [대시보드] [회의 목록] [회의 생성] [안건 관리] [검색] │  ← 상단 네비게이션
├─────────────────────────────────────────────────────┤
│                                                     │
│              [ 메인 콘텐츠 영역 ]                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 2.2 화면별 상세 구성

#### ① 대시보드 (홈)
- 이번 주 예정 회의 카드 (날짜·참석자·상태)
- 미완료 액션 아이템 목록
- 최근 완료된 회의록 요약
- 나의 담당 액션 아이템 현황

#### ② 회의 목록 페이지
- 주간/월간 캘린더 뷰 + 리스트 뷰 전환
- 상태 필터: `예정` / `진행 중` / `완료` / `취소`
- 부서·프로젝트별 필터링
- 검색 (제목, 참석자, 안건 키워드)

#### ③ 회의 생성/편집 페이지
- **기본 정보 입력**
  - 회의명, 일시, 장소(또는 온라인 링크)
  - 참석자 초대 (사내 디렉토리 연동)
  - 회의 유형 선택 (주간보고/프로젝트/긴급 등)
- **안건 등록 (사전)**
  - 안건 제목 + 담당자 + 예상 소요시간
  - 관련 첨부파일 업로드
  - 안건 순서 드래그앤드롭 정렬
- **사전 배포**: 참석자에게 안건 사전 공유 알림

#### ④ 회의 진행 (실시간 작성) 페이지
- **타이머**: 회의 시간 측정 (안건별 시간 분배)
- **안건별 패널**
  - 안건 내용 표시
  - 검토의견 입력창 (참석자별 실시간 입력 또는 사회자 대리 입력)
  - 결정 사항 기록란
  - 액션 아이템 등록 (담당자 + 기한 + 우선순위)
- **회의 진행 상태**: 현재 안건 표시, 다음 안건 이동 버튼

#### ⑤ 회의록 정리/완료 페이지
- 자동 생성된 회의록 미리보기
- 항목별 수정 편집 (리치 텍스트 에디터)
- 결재/승인 워크플로우 (작성자 → 팀장 → 배포)
- **배포 옵션**: 이메일 발송 / 사내 포털 게시 / PDF 다운로드

#### ⑥ 안건 관리 페이지
- 모든 회의의 안건 통합 조회
- 안건별 결정 이력 추적
- 미결 안건 하이라이트

#### ⑦ 액션 아이템 트래커
- 담당자별·기한별 필터
- 진행 상태 업데이트 (대기 → 진행중 → 완료)
- 기한 초과 알림 표시

## 4. 데이터 구조 설계

### 4.1 주요 엔티티

```
회의 (Meeting)
├── 기본정보: ID, 제목, 일시, 장소, 유형, 상태
├── 참석자 목록 (MeetingParticipant)
├── 안건 목록 (AgendaItem)
│   ├── 안건 제목, 담당자, 소요시간, 순서
│   ├── 검토의견 (ReviewComment)
│   │   └── 작성자, 내용, 작성시간
│   ├── 결정사항 (Decision)
│   └── 액션아이템 (ActionItem)
│       └── 담당자, 기한, 우선순위, 상태
├── 첨부파일 (Attachment)
└── 회의록 메타 (승인상태, 배포일시)
```

### 4.2 상태 흐름

```
[회의 상태]
초안 → 안건공유완료 → 진행중 → 회의록작성중 → 검토중 → 배포완료

[액션아이템 상태]
등록 → 진행중 → 완료 / 취소 / 기한초과
```

## 6. 기술 스택 (권장안)

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| Frontend | React + TypeScript | 컴포넌트 재사용, 타입 안전성 |
| UI 라이브러리 | Tailwind CSS + shadcn/ui | 빠른 UI 구성, 커스터마이징 용이 |
| 상태 관리 | Zustand 또는 React Query | 서버 상태 캐싱 최적화 |
| Backend | Python (FastAPI) | API 개발 속도 |
| Database | PostgreSQL 15 | 관계형 데이터 구조에 적합 |
| 인증 | JWT + OAuth (사내 SSO 연동) | 보안 및 편의성 |
| 파일 저장 | S3 또는 사내 스토리지 | 첨부파일 관리 |
| 알림 | 이메일 (SMTP) + WebSocket | 실시간 알림 |

## 8. 성공 지표 (KPI)

- 📊 회의록 작성 완료율 **90% 이상** (회의 후 24시간 내)
- ⚡ 회의록 작성 시간 **50% 단축** (기존 대비)
- ✅ 액션 아이템 기한 내 완료율 **80% 이상**
- 👥 시스템 주간 활성 사용자 **전체 구성원 85% 이상**
- 🔍 과거 회의록 재참조 빈도 증가 (분기별 측정)

## 9. 추가 제안 사항

1. **온보딩 튜토리얼**: 첫 회의 생성 시 단계별 가이드 제공
2. **모바일 반응형**: 태블릿/모바일에서도 실시간 입력 가능하도록 설계
3. **정기 리포트**: 주간·월간 회의 통계 자동 이메일 발송
4. **피드백 채널**: 시스템 개선 의견 수집 기능 내장
5. **오프라인 대응**: 인터넷 불안정 시 로컬 임시 저장 후 자동 동기화
